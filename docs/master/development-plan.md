# Master Development Plan

## From development to go-live
- Create repo and baseline standards
- Set up CI, tests, lint, type-checking
- Build auth and tenant controls
- Build structured domain model
- Build ingestion and workflow engine
- Build stakeholder views
- Add AI with bounded tools
- Add monitoring, audit logs, and support playbooks
- Pilot, fix, harden, and launch

## Go-live preconditions
- Security review complete
- UAT passed
- Monitoring active
- Backup and restore tested
- Support runbook approved
- Release checklist signed off
